## Add `UIProvider`

After installing Yamada UI, add the `UIProvider`.

```tsx live=false title=_app.tsx highlight={6,8}
import type { AppProps } from 'next/app'
import { UIProvider } from '@yamada-ui/react'

export default function App({ Component, pageProps }: AppProps) {
  return (
    <UIProvider>
      <Component {...pageProps} />
    </UIProvider>
  )
}
```

:::note
The [Default theme](/styled-system/theming/default-theme) resides within the `UIProvider`.
:::

:::note
For theme and configuration personalization, consult [Customize the theme](/styled-system/theming/customize-theme) and [Customize the configuration](/styled-system/configure/customize-config).
:::

## Incorporate `ColorModeScript`

To ensure the color mode functions correctly, you must incorporate the `ColorModeScript` within the `body`.

This is crucial because the color mode leverages `localStorage` or `cookies`, ensuring synchronous operation upon page loading.

```tsx live=false title=_document.tsx highlight={10}
import { Html, Head, Main, NextScript } from 'next/document'
import { ColorModeScript, defaultConfig } from '@yamada-ui/react'

export default function Document() {
  return (
    <Html lang='en'>
      <Head />

      <body>
        <ColorModeScript initialColorMode={defaultConfig.initialColorMode} />
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

:::note status=warning
Ensure that the `initialColorMode` prop matches the `initialColorMode` in your config. If you have a specific config, provide its `initialColorMode`.
:::

## Use the components

After integrating the `UIProvider`, you can invoke the components within the application.

```tsx live=false title=index.tsx
import { Button } from '@yamada-ui/react'

export default function Home() {
  return <Button>Click me!</Button>
}
```
