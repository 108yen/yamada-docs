## Install the package

First, create an application.

Run one of the following commands in the terminal.

<PackageManagers

    npm: "npx create-next-app my-app --typescript",

/>

## Adding `UIProvider`

After installing Yamada UI, add `UIProvider`.

```tsx live=false title=_app.tsx highlight={6,8}
import type { AppProps } from "next/app"
import { UIProvider } from "@yamada-ui/react"

export default function App({ Component, pageProps }: AppProps) {
  return (
    <UIProvider>
      <Component {...pageProps} />
    </UIProvider>
  )
}
```

:::note
The [default theme](/styled-system/theming/default-theme) is included in `UIProvider`.
:::

:::note
If you want to customize the theme or config, please check [customize the theme](/styled-system/theming/customize-theme) and [customize the config](/styled-system/configure/customize-config).
:::

## Add `ColorModeScript`

In order for the color mode to function properly, you need to add `ColorModeScript` within the `body`.

The reason is that the color mode is implemented using `localStorage` or `cookies`, and it is necessary to synchronize correctly at the time of page loading.

```tsx live=false title=_document.tsx highlight={10}
import { Html, Head, Main, NextScript } from "next/document"
import { ColorModeScript, defaultConfig } from "@yamada-ui/react"

export default function Document() {
  return (
    <Html lang="en">
      <Head />

      <body>
        <ColorModeScript initialColorMode={defaultConfig.initialColorMode} />
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

:::note status=warning
Please pass the same `initialColorMode` as the config to the `initialColorMode` of `props`. If you have your own config, please pass its `initialColorMode`.
:::

## Using the Component

Once you've added the `UIProvider`, call the component within your application.

```tsx live=false title=index.tsx
import { Button } from "@yamada-ui/react"

export default function Home() {
  return <Button>Click me!</Button>
}
```
