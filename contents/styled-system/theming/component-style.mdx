Maintaining flexible styles for components in a growing project is a challenging task.

To solve this, Yamada UI constructs each component with basic styles and modifier styles. This allows for flexible style changes.

In addition, we provide `useComponentStyle` and `useMultiComponentStyle` to manage the styles of the components you create with themes.

## Basic Styles and Modifier Styles

Most components are composed of basic styles and modifier styles that change visuals and sizes based on properties.

Common modifier styles include the following:

- `Size`: Components can have various sizes (small, medium, large, etc.).
- `Variant`: Components can have various visuals (outline, solid, ghost, etc.).
- `Color Scheme`: For a specific variant, components can have different color schemes.
- `Color Mode`: Components can change styles based on the color mode (light, dark).

Also, there is an order in which styles are set.

The order is as follows, and the same property values are overwritten:

1. `Basic Style`
2. `Size`
3. `Variant`

:::note status=warning
`Color Scheme` and `Color Mode` are values that assist modifier styles, not styles themselves.
:::

## Single Part and Multiple Part Components

Yamada UI components consist of **components made up of a single part** (such as `Button`, `Link`, etc.) and **components made up of multiple parts** (such as `Tab`, `Menu`, etc.).

### Single Part Component

A single part component is a component composed of a single element. For example, the `Button` component renders the `button` HTML element.

```tsx live=false
<Button>Click me!</Button>
```

:::note status=warning
Even though it may appear as a **single part component**, there are cases where it is a **multi-part component** that internally switches elements based on conditional branching. For example, `Tag`, `Checkbox`, `Radio`, etc. are applicable.
:::

### Component with Multiple Parts

A component with multiple parts is a component that consists of multiple elements. The child elements are dependent on the parent element. For example, the `List` component is composed of `ListItem` and `ListIcon`.

```tsx live=false
<List>
  <ListItem>Shall I dance for joy if you don't mind?</ListItem>
  <ListItem>Take my life!!! Toss!!!</ListItem>
  <ListItem>U... Behind me, who boasts the fastest speed in the universe...</ListItem>
  <ListItem>There was no way we could win...</ListItem>
  <ListItem>Am I like a trash picker at the game venue...</ListItem>
</List>
```

### Styling of Single Part Components

:::note status=warning
Before styling, please refer to [Theme Operation](/styled-system/theming/customize-theme#Theme Operation) and [Changing the Style of Components](/styled-system/theming/customize-theme#Changing the Style of Components).
:::

[Changing the Style of Components](/styled-system/theming/customize-theme#Changing the Style of Components) was styling that inherited the default theme.

This time, we will style the `Button` anew.

```ts title=./theme/components/button.ts
export const Button: ComponentStyle = {
  // Basic style
  baseStyle: {
    lineHeight: 1.2,
    fontWeight: 'semibold',
    transitionProperty: 'common',
    transitionDuration: 'slower',
  },

  // Visual style
  variants: {
    solid: {
      bg: ['gray.500', 'gray.200'],
      color: ['white', 'gray.800'],
    },
    outline: {
      borderWidth: '1px',
    },
  },

  // Size style
  sizes: {
    sm: {
      h: 8,
      minW: 8,
      fontSize: 'sm',
      px: 3,
    },
    md: {
      h: 10,
      minW: 10,
      fontSize: 'md',
      px: 4,
    },
    lg: {
      h: 12,
      minW: 12,
      fontSize: 'lg',
      px: 6,
    },
  },

  // Default values for `variant` and `size`, etc.
  defaultProps: {
    variant: 'solid',
    size: 'md',
  },
}
```

Next, we will support dynamic `color schemes`. To receive a `color scheme` from a component, use a function.

```ts title=./theme/components/button.ts highlight={12,13,14,15,16,17,18,19,48}
export const Button: ComponentStyle = {
  // Basic style
  baseStyle: {
    lineHeight: 1.2,
    fontWeight: 'semibold',
    transitionProperty: 'common',
    transitionDuration: 'slower',
  },

  // Visual style
  variants: {
    solid: ({ colorScheme: c }) => ({
      bg: [`${c}.500`, `${c}.200`],
      color: ['white', `${c}.800`],
    }),
    outline: ({ colorScheme: c }) => ({
      borderWidth: '1px',
      borderColor: [`${c}.600`, `${c}.300`],
    }),
  },

  // Size style
  sizes: {
    sm: {
      h: 8,
      minW: 8,
      fontSize: 'sm',
      px: 3,
    },
    md: {
      h: 10,
      minW: 10,
      fontSize: 'md',
      px: 4,
    },
    lg: {
      h: 12,
      minW: 12,
      fontSize: 'lg',
      px: 6,
    },
  },

  // Default values for `variant` and `size`, etc.
  defaultProps: {
    variant: 'solid',
    size: 'md',
    colorScheme: 'gray',
  },
}
```

:::note
Functions can retrieve `theme`, `colorMode`, and `props` in addition to `colorScheme`.
:::

Set the style of the created component to the theme.

```ts title=./theme/components/index.ts
import { Button } from './button'

export default { Button }
```

```ts title=./theme/index.ts
import { extendTheme, defaultTheme, UsageTheme } from '@yamada-ui/react'
import customComponents from './components'

const defaultComponents = defaultTheme.components

const customTheme: UsageTheme = {
  components: { ...defaultComponents, ...customComponents },
}

export default extendTheme(customTheme)({ omit: ['components'] })
```

### Styling Multi-part Components

:::note status=warning
Before styling, please refer to [Theme Operation](/styled-system/theming/customize-theme#Theme Operation) and [Changing Component Styles](/styled-system/theming/customize-theme#Changing Component Styles).
:::

[Changing Component Styles](/styled-system/theming/customize-theme#Changing Component Styles) was styling that inherited the default theme.

This time, we will style the `List` anew.

Multi-part components are styled for each `key` associated with each part.

```ts title=./theme/components/list.ts
export const List: ComponentMultiStyle = {
  // Basic style
  baseStyle: {
    container: {},
    item: {
      lineHeight: 1.2,
      rounded: 'md',
    },
    icon: {
      me: '2',
      display: 'inline-block',
      verticalAlign: 'middle',
    },
  },

  // Visual style
  variants: {
    solid: {
      bg: ['gray.500', 'gray.200'],
      color: ['white', 'gray.800'],
    },
    outline: {
      borderWidth: '1px',
    },
    unstyled: { rounded: 'inherit', lineHeight: 'inherit' },
  },

  // Size style
  sizes: {
    sm: {
      container: { gap: 'sm' },
      item: { fontSize: 'sm' },
    },
    md: {
      container: { gap: 'md' },
      item: { fontSize: 'md' },
    },
    lg: {
      container: { gap: 'lg' },
      item: { fontSize: 'lg' },
    },
  },

  // Default values for `variant`, `size`, etc.
  defaultProps: {
    variant: 'solid',
    size: 'md',
  },
}
```

Next, we will support dynamic `color schemes`. To receive a `color scheme` from a component, use a function.

```ts title=./theme/components/list.ts highlight={18,19,20,21,22,23,47}
export const List: ComponentMultiStyle = {
  // Basic style
  baseStyle: {
    container: {},
    item: {
      lineHeight: 1.2,
      rounded: 'md',
    },
    icon: {
      me: '2',
      display: 'inline-block',
      verticalAlign: 'middle',
    },
  },

  // Visual style
  variants: {
    solid: ({ colorScheme: c }) => ({
      item: { bg: [`${c}.500`, `${c}.200`], color: ['white', `${c}.800`] },
    }),
    outline: ({ colorScheme: c }) => ({
      item: { borderWidth: '1px', borderColor: [`${c}.600`, `${c}.300`] },
    }),
    unstyled: { item: { rounded: 'inherit', lineHeight: 'inherit' } },
  },

  // Size style
  sizes: {
    sm: {
      container: { gap: 'sm' },
      item: { h: 8, minW: 8, fontSize: 'sm', px: 3 },
    },
    md: {
      container: { gap: 'md' },
      item: { h: 10, minW: 10, fontSize: 'md', px: 4 },
    },
    lg: {
      container: { gap: 'lg' },
      item: { h: 12, minW: 12, fontSize: 'lg', px: 6 },
    },
  },

  // Default values for `variant`, `size`, etc.
  defaultProps: {
    variant: 'solid',
    size: 'md',
    colorScheme: 'gray',
  },
}
```

:::note status=warning
`key` varies depending on the component. To check the `key` of each component, please refer to the `theme` on the component's page.
:::

Set the style of the created component to the theme.

```ts title=./theme/components/index.ts
import { List } from './list'

export default { List }
```

```ts title=./theme/index.ts
import { extendTheme, defaultTheme, UsageTheme } from '@yamada-ui/react'
import customComponents from './components'

const defaultComponents = defaultTheme.components

const customTheme: UsageTheme = {
  components: { ...defaultComponents, ...customComponents },
}

export default extendTheme(customTheme)({ omit: ['components'] })
```

## Creating Custom Components

Depending on the project, you may need to create your own components. To manage the style of these custom components with a theme, Yamada UI provides `useComponentStyle` and `useMultiComponentStyle` to get the style of the components.

This time, we will create a single part component called `Banner`.

First, create the component.

```tsx live=false
import { FC } from 'react'
import { ui, HTMLUIProps, ThemeProps } from '@yamada-ui/react'

type BannerOptions = {}

export type BannerProps = HTMLUIProps<'button'> & ThemeProps<'Banner'> & BannerOptions

export const Banner: FC<BannerProps> = (props) => {
  return <ui.button {...props} />
}
```

Next, set the style of `Banner` in the theme.

```ts title=./theme/components/banner.ts
export const Banner: ComponentStyle = {
  baseStyle: {
    // Basic style
  },

  variants: {
    // Visual style
  },

  sizes: {
    // Size style
  },

  defaultProps: {
    // Default values such as `variant` and `size`
  },
}
```

```ts title=./theme/components/index.ts
import { Banner } from './banner'

export default { Banner }
```

```ts title=./theme/index.ts
import { extendTheme, defaultTheme, UsageTheme } from '@yamada-ui/react'
import customComponents from './components'

const defaultComponents = defaultTheme.components

const customTheme: UsageTheme = {
  components: { ...defaultComponents, ...customComponents },
}

export default extendTheme(customTheme)({ omit: ['components'] })
```

:::note
It is recommended to perform type completion for the created `variants` and `sizes`. Type completion is done using the [CLI](/styled-system/cli).
:::

Finally, call `useComponentStyle` within the component and load the style.

```tsx live=false highlight={2,9,10,12}
import { FC } from 'react'
import { ui, HTMLUIProps, ThemeProps, useComponentStyle, omitThemeProps } from '@yamada-ui/react'

type BannerOptions = {}

export type BannerProps = HTMLUIProps<'button'> & ThemeProps<'Banner'> & BannerOptions

export const Banner: FC<BannerProps> = (props) => {
  const [styles, mergedProps] = useComponentStyle('Banner', props)
  const rest = omitThemeProps(mergedProps)

  return <ui.button __css={styles} {...rest} />
}
```

If `Banner` was a multi-part component, call `useMultiComponentStyle` within the component and the style reference would be as follows.

```tsx live=false highlight={6,7,15,16,19,20,21,22}
import { FC } from 'react'
import {
  ui,
  HTMLUIProps,
  ThemeProps,
  useMultiComponentStyle,
  omitThemeProps,
} from '@yamada-ui/react'

type BannerOptions = {}

export type BannerProps = HTMLUIProps<'button'> & ThemeProps<'Banner'> & BannerOptions

export const Banner: FC<BannerProps> = (props) => {
  const [styles, mergedProps] = useMultiComponentStyle('Banner', props)
  const rest = omitThemeProps(mergedProps)

  return (
    <ui.button __css={styles.container} {...rest}>
      <ui.span __css={styles.icon} />
      {children}
    </ui.button>
  )
}
```

:::note status=warning
Please note that the style is passed to `__css`. This is an API like `sx`, but the style priority is the lowest. `__css` is a `props` used internally in Yamada UI components.
:::
