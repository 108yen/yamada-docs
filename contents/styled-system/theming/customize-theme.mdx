title: Customizing the Theme

---

All components of Yamada UI inherit styles and tokens from the theme. Depending on the scenario, you may need to customize things like `color` to match your project.

:::note status=warning
When customizing the theme, we recommend using `extendTheme`. This creates a theme that inherits from the [default theme](/styled-system/theming/default-theme). If you do not use `extendTheme`, it will become a completely new theme that does not inherit from the [default theme](/styled-system/theming/default-theme).
:::

## Theme Operation

If you customize a theme, it is recommended to perform type completion. Type completion uses the [CLI](/styled-system/cli).

:::note status=warning
When using `CLI`, you need to create a theme file because it refers to the theme that is exported by default in the file.
:::

Considering that theme tokens and styles may be added or changed during development, it is recommended to keep the theme in a folder.

```sh
./theme
├ components # Folder to define the style of each component
├ tokens # Folder to define each token
├ styles # Folder to define global styles and reset styles
├ config.ts # File to set the config
└ index.ts # File to export the theme by default
```

```ts title=./theme/index.ts
import { extendTheme, UsageTheme } from '@yamada-ui/react'
// import styles from './styles'
// import components from './components'
// import tokens from './tokens'

const customTheme: UsageTheme = {
  // styles,
  // components,
  // ...tokens,
}

export default extendTheme(customTheme)()
```

:::note
`extendTheme` is used to export the theme by default.
:::

## Changing Tokens

This time, let's try changing new and existing tokens in `colors`.

Create `colors.ts` under `tokens`.

```ts title=./theme/tokens/colors.ts
import { ThemeTokens } from '@yamada-ui/react'

export const colors: ThemeTokens = {
  banner: '#9d38a0',
  black: ['#1F2123', '#101112'],
}
```

:::note
We have added a new `banner` to `colors` and changed `black`. Also, by defining an array, it can support color modes.
:::

Create `index.ts` and export the `colors` you created earlier.

```ts title=./theme/tokens/index.ts
import { colors } from './colors'

export default { colors }
```

Include `tokens` in `customTheme`.

```ts title=./theme/index.ts highlight={4,9}
import { extendTheme, UsageTheme } from '@yamada-ui/react'
// import styles from './styles'
// import components from './components'
import tokens from './tokens'

const customTheme: UsageTheme = {
  // styles,
  // components,
  ...tokens,
}

export default extendTheme(customTheme)()
```

If you don't want to inherit only `colors` from the default theme, pass `omit` to `extendTheme`.

```ts title=./theme/index.ts highlight={12}
import { extendTheme, UsageTheme } from '@yamada-ui/react'
// import styles from './styles'
// import components from './components'
import tokens from './tokens'

const customTheme: UsageTheme = {
  // styles,
  // components,
  ...tokens,
}

export default extendTheme(customTheme)({ omit: ['colors'] })
```

## Changing Semantic Tokens

Within the semantic tokens, a color scheme used in each component is set.

Here are the actual values that are set.

```ts
export const semantics: ThemeSemantics = {
  colors: {
    primary: 'blue.500',
    secondary: 'purple.500',
    success: 'emerald.500',
    warning: 'orange.500',
    danger: 'red.500',
    link: 'sky.500',
  },
  colorSchemes: {
    primary: 'blue',
    secondary: 'purple',
    success: 'emerald',
    warning: 'orange',
    danger: 'red',
    link: 'sky',
  },
}
```

This time, let's try changing the default color scheme.

Create `semantics.ts` under `theme`.

```ts title=./theme/semantics.ts
import { ThemeSemantics } from '@yamada-ui/react'

export const semantics: ThemeSemantics = {
  colors: {
    primary: 'pink.500',
  },
  colorSchemes: {
    primary: 'pink',
  },
}
```

Include `semantics` in `customTheme`.

```ts title=./theme/index.ts highlight={5,10}
import { extendTheme, UsageTheme } from '@yamada-ui/react'
// import styles from './styles'
// import components from './components'
import tokens from './tokens'
import { semantics } from './semantics'

const customTheme: UsageTheme = {
  // styles,
  // components,
  semantics,
  ...tokens,
}

export default extendTheme(customTheme)()
```

:::note
If you want to learn more about semantic tokens, please see [here](/styled-system/semantic-tokens).
:::

## Changing Styles

You can set [globalStyle](/styled-system/global-styles), [resetStyle](/styled-system/reset-styles), [layerStyles](/styled-system/text-and-layer-styles), [textStyles](/styled-system/text-and-layer-styles), etc. in `styles`.

This time, let's try changing the `globalStyle` that sets styles such as the `body`.

Create `global-style.ts` under `styles`.

```ts title=./theme/styles/global-style.ts
import { UIStyle } from '@yamada-ui/react'

export const globalStyle: UIStyle = {
  body: {
    bg: ['black', 'white'],
  },
}
```

:::note
The background of the `body` is being changed. Also, by defining an array, you can support color modes.
:::

Create `index.ts` and export the `globalStyle` you created earlier.

```ts title=./theme/styles/index.ts
import { globalStyle } from './global-style'

export default { globalStyle }
```

Include `styles` in `customTheme`.

```ts title=./theme/index.ts highlight={2,8}
import { extendTheme, UsageTheme } from '@yamada-ui/react'
import styles from './styles'
// import components from './components'
import tokens from './tokens'
import { semantics } from './semantics'

const customTheme: UsageTheme = {
  styles,
  // components,
  semantics,
  ...tokens,
}

export default extendTheme(customTheme)()
```

If you don't want to inherit only `styles` from the default theme, pass `omit` to `extendTheme`.

```ts title=./theme/index.ts highlight={12}
import { extendTheme, UsageTheme } from '@yamada-ui/react'
import styles from './styles'
// import components from './components'
import tokens from './tokens'
import { semantics } from './semantics'

const customTheme: UsageTheme = {
  styles,
  // components,
  semantics,
  ...tokens,
}

export default extendTheme(customTheme)({ omit: ['styles'] })
```

:::note
If you want to learn about text and layer styles, please see [here](/styled-system/text-and-layer-styles).
:::

## Changing the Style of Components

Depending on the project, you may want to change the style of Yamada UI's components. Since Yamada UI's components refer to the style of the theme, you can easily change the style of the components.

The style of a component is composed of the component's basic style (`baseStyle`), styles of various sizes (`sizes`), styles of various visuals (`variants`), and default `props` (`defaultProps`).

:::note
Not all components have a `size` or `variant`. To check which components are compatible, refer to the `props` on the component's page.
:::

:::note
Yamada UI can style components by setting style `props` on the component. However, large projects have many developers, and setting styles individually can compromise design consistency. In that case, you can avoid this by setting the common style in the style of the theme's components.
:::

This time, let's change the `Container` component.

Create `container.ts` under `components`.

```ts title=./theme/components/container.ts
import { ComponentStyle } from '@yamada-ui/react'

export const Container: ComponentStyle = {
  baseStyle: {
    bg: 'green.100',
  },
  sizes: {
    sm: { p: 'sm', fontSize: 'sm' },
    md: { p: 'md', fontSize: 'md' },
    lg: { p: 'lg', fontSize: 'lg' },
  },
  variants: {
    'with-border-solid': {
      borderWidth: '1px',
    },
    'with-border-dotted': {
      borderWidth: '1px',
      borderStyle: 'dotted',
    },
  },
  defaultProps: {
    size: 'md',
    variant: 'with-border-solid',
  },
}
```

:::note
The background color has been changed, and each size and variant is defined in `sizes` and `variants`, respectively. `defaultProps` sets the initial values.
:::

Create `index.ts` and export the `Container` you just created.

```ts title=./theme/components/index.ts
import { Container } from './container'

export default { Container }
```

Include `components` in `customTheme`.

```ts title=./theme/index.ts highlight={3,9}
import { extendTheme, UsageTheme } from '@yamada-ui/react'
import styles from './styles'
import components from './components'
import tokens from './tokens'
import { semantics } from './semantics'

const customTheme: UsageTheme = {
  styles,
  components,
  semantics,
  ...tokens,
}

export default extendTheme(customTheme)()
```

Now, the background color of `Container` has been changed, and the style can be changed by passing `size` or `variant`.

```tsx live=false
import { Container } from '@yamada-ui/react'

const Demo = () => {
  return (
    <Container size='sm' variant='with-border-dotted'>
      This is Container
    </Container>
  )
}
```

:::note status=warning
The `Container` changed this time is a component composed of a single element. There are also components composed of multiple elements (such as `Accordion`). When changing a component composed of multiple elements, the type definition becomes `ComponentMultiStyle`.
:::

:::note
To check the theme style of each component, refer to the `Theme` on the component's page.
:::

:::note
You can create styles for custom components, not just the components provided by Yamada UI. For more details, please see [here](/styled-system/theming/component-style#creating-custom-components).
:::

## Utility

Yamada UI provides many theme utilities.

### Using `extendTheme`

Create a new theme by inheriting the default theme. It is also possible to pass multiple objects and specify options.

#### Merging multiple themes

```ts title=./theme/index.ts highlight={12}
import { extendTheme, UsageTheme } from '@yamada-ui/react'

const firstTheme: UsageTheme = {}
const secondTheme: UsageTheme = {}

export default extendTheme(firstTheme, secondTheme)()
```

#### Not inheriting from the default theme

```ts title=./theme/index.ts highlight={12}
import { extendTheme, UsageTheme } from '@yamada-ui/react'

const firstTheme: UsageTheme = {}
const secondTheme: UsageTheme = {}

export default extendTheme(firstTheme, secondTheme)({ merge: false })
```

#### Inheriting from the default theme excluding specific tokens

```ts title=./theme/index.ts highlight={12}
import { extendTheme, UsageTheme } from '@yamada-ui/react'

const customTheme: UsageTheme = {}

export default extendTheme(customTheme)({ omit: ['colors'] })
```

#### Inheriting only specific tokens from the default theme

```ts title=./theme/index.ts highlight={12}
import { extendTheme, UsageTheme } from '@yamada-ui/react'

const customTheme: UsageTheme = {}

export default extendTheme(customTheme)({ pick: ['spaces'] })
```

### Using `extendBaseTheme`

Create a new theme that inherits only the global styles and tokens such as colors from the default theme. It is also possible to pass multiple objects and specify options.

```ts title=./theme/index.ts highlight={12}
import { extendBaseTheme, UsageTheme } from '@yamada-ui/react'
import styles from './styles'
import tokens from './tokens'

const customTheme: UsageTheme = {
  styles,
  ...tokens,
}

export default extendBaseTheme(customTheme)()
```

:::note
`extendBaseTheme` internally uses `baseTheme`. `baseTheme` has defined tokens such as global styles and colors, but the styles of the components are not defined.
:::

### Using `extendStyle`

Inherit `globalStyle` or `resetStyle` to create a new style.

```ts
import { extendStyle, UIStyle } from '@yamada-ui/react'

const globalStyle: UIStyle = extendStyle('globalStyle', {
  /**
   * Define additional styles
   */
})
```

### Using `extendToken`

Inherits a specific token to create a new token.

```ts
import { extendToken, ThemeTokens } from '@yamada-ui/react'

const colors: ThemeTokens = extendToken('colors', {
  /**
   * Define additional tokens
   */
})
```

### Using `extendComponent`

Inherits the style of a specific component to create the style of a new component.

```ts
import { extendComponent, ComponentStyle } from '@yamada-ui/react'

const Button: ComponentStyle = extendComponent('Button', {
  /**
   * Define additional `baseStyle` and `sizes`
   */
})
```

### Using `extendComponentSize`

This inherits the size of a specific component to create the size of a new component.

```ts
import { extendComponentSize, ComponentStyle } from '@yamada-ui/react'

const Tag: ComponentStyle = {
  baseStyle: {
    /**
     * Define a new style
     */
  },
  sizes: extendComponentSize('Tag', {
    /**
     * Define additional sizes
     */
  }),
}
```

:::note
This function can be used in cases where you only want to inherit `sizes`, but do not want to inherit `baseStyle` or `variants`.
:::

### Using `extendComponentVariant`

This will inherit a specific component's variant and create a new component's variant.

```ts
import { extendComponentVariant, ComponentStyle } from '@yamada-ui/react'

const Tag: ComponentStyle = {
  baseStyle: {
    /**
     * Define new styles
     */
  },
  variants: extendComponentVariant('Tag', {
    /**
     * Define additional variants
     */
  }),
}
```

:::note
This function can be used in cases where you only want to inherit `variants`, but do not want to inherit `baseStyle` or `sizes`.
:::

### Using `extendComponentDefaultProps`

This extends the `props` of a specific component to create the `props` of a new component.

```ts
import { extendComponentDefaultProps, ComponentStyle } from '@yamada-ui/react'

const Tag: ComponentStyle = {
  baseStyle: {
    /**
     * Define a new style
     */
  },
  defaultProps: extendComponentDefaultProps('Tag', {
    /**
     * Define additional props
     */
  }),
}
```

:::note
This function can be used in cases where you want to inherit only `defaultProps`, but not `baseStyle` or `sizes`.
:::

### Using `withDefaultSize`

Change the default size of `all` or `some` components.

```ts title=./theme/index.ts
import { extendTheme, withDefaultSize } from '@yamada-ui/react'

/**
 * omitted
 */

export default extendTheme(
  customTheme,
  withDefaultSize({
    size: 'lg',
    components: ['Badge', 'Tag', 'Button'], // If you pass an empty array, it will be set for all components.
  }),
)()
```

### Using `withDefaultVariant`

This changes the default variant of `all` or `some` components.

```ts title=./theme/index.ts
import { extendTheme, withDefaultVariant } from '@yamada-ui/react'

/**
 * omitted
 */

export default extendTheme(
  customTheme,
  withDefaultVariant({
    variant: 'solid',
    components: ['Badge', 'Tag', 'Button'], // If you pass an empty array, it will be set for all components.
  }),
)()
```

### Using `withDefaultColorScheme`

This changes the default color scheme for `all` or `some` of the components.

```ts title=./theme/index.ts
import { extendTheme, withDefaultColorScheme } from '@yamada-ui/react'

/**
 * omitted
 */

export default extendTheme(
  customTheme,
  withDefaultColorScheme({
    colorScheme: 'primary',
    components: ['Badge', 'Tag', 'Button'], // If you pass an empty array, it will be set for all components.
  }),
)()
```

### Using `withDefaultProps`

This changes all or some of the default `props` of the components in bulk.

```ts title=./theme/index.ts
import { extendTheme, withDefaultProps } from '@yamada-ui/react'

/**
 * omitted
 */

export default extendTheme(
  customTheme,
  withDefaultProps({
    defaultProps: {
      size: 'lg',
      variant: 'solid',
      colorScheme: 'primary',
    },
    components: ['Badge', 'Tag', 'Button'], // If you pass an empty array, it will be set for all components.
  }),
)()
```
