By customizing the config, you can change the [color mode](/styled-system/color-mode) and the prefix of `CSS` custom properties.

:::note status=warning
When customizing the config, we recommend using `extendConfig`. This creates a config that inherits from the [default config](/styled-system/configure/default-config). If you do not use `extendConfig`, it will become a completely new config that does not inherit from the [default config](/styled-system/configure/default-config).
:::

## Change Color Mode

Use `extendConfig` to pass the generated config to `UIProvider`'s `config`.

```tsx live=false highlight={3,7}
import { UIProvider, extendConfig } from '@yamada-ui/react'

const customConfig = extendConfig({ initialColorMode: 'system' })

const App = () => {
  return (
    <UIProvider config={customConfig}>
      <YourApplication />
    </UIProvider>
  )
}
```

## Changing the prefix of `CSS` custom properties

Use `extendConfig` to pass the generated config to the `config` of `UIProvider`.

```tsx live=false highlight={3,7}
import { UIProvider, extendConfig } from '@yamada-ui/react'

const customConfig = extendConfig({ var: { prefix: 'hoge' } })

const App = () => {
  return (
    <UIProvider config={customConfig}>
      <YourApplication />
    </UIProvider>
  )
}
```
